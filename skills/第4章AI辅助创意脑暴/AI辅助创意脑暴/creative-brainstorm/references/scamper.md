# SCAMPER 创意发散方法

SCAMPER 是经典的创意激发框架，通过 7 个维度强制发散思维。

## 七个维度详解

### S - Substitute（替代）

**核心问题**：什么可以被替换？

思考方向：
- 材料/成分可以换成什么？
- 人员/角色可以换成谁？
- 流程/步骤可以换成什么？
- 技术/工具可以换成什么？
- 时间/地点可以换成什么？

示例：
- 用 AI 替代人工客服
- 用订阅制替代一次性购买
- 用线上替代线下

### C - Combine（组合）

**核心问题**：可以和什么结合？

思考方向：
- 可以和哪些功能结合？
- 可以和哪些产品结合？
- 可以和哪些服务结合？
- 可以和哪些场景结合？
- 可以和哪些用户群结合？

示例：
- 学习 + 游戏 = 游戏化学习
- 社交 + 电商 = 社交电商
- 健身 + 音乐 = 运动节拍 App

### A - Adapt（适应/借鉴）

**核心问题**：可以借鉴什么？

思考方向：
- 其他行业怎么解决类似问题？
- 历史上有什么可借鉴的？
- 自然界有什么可模仿的？
- 其他国家/地区怎么做？
- 竞品有什么值得学习的？

示例：
- 借鉴 Uber 模式做家政服务
- 借鉴游戏成就系统做员工激励
- 借鉴蚂蚁协作做分布式系统

### M - Modify（修改/放大/缩小）

**核心问题**：可以放大或缩小什么？

思考方向：
- 功能可以增加/减少吗？
- 规模可以扩大/缩小吗？
- 频率可以提高/降低吗？
- 价格可以提高/降低吗？
- 时间可以延长/缩短吗？

示例：
- 极简版 vs 专业版
- 个人版 vs 企业版
- 日更 vs 周更 vs 月更

### P - Put to other use（他用）

**核心问题**：还能用在哪里？

思考方向：
- 还能解决什么其他问题？
- 还能服务什么其他人群？
- 还能应用在什么其他场景？
- 副产品可以怎么利用？
- 失败的尝试可以怎么转化？

示例：
- 直播技术用于在线教育
- 游戏引擎用于建筑可视化
- 电商物流用于同城配送

### E - Eliminate（消除）

**核心问题**：可以去掉什么？

思考方向：
- 哪些功能是多余的？
- 哪些步骤可以省略？
- 哪些中间环节可以去掉？
- 哪些限制可以打破？
- 哪些传统可以抛弃？

示例：
- 去掉中间商（D2C）
- 去掉注册流程（游客模式）
- 去掉实体店（纯线上）

### R - Reverse（逆向/重排）

**核心问题**：可以颠倒什么？

思考方向：
- 顺序可以颠倒吗？
- 角色可以互换吗？
- 因果可以反转吗？
- 主次可以调换吗？
- 内外可以翻转吗？

示例：
- 用户变成生产者（UGC）
- 先用后付（BNPL）
- 买家定价（逆向拍卖）

## 使用技巧

1. **不要跳过任何维度**：即使某个维度看起来不适用，也要尝试思考
2. **每个维度至少 2 个想法**：强制输出，避免思维惰性
3. **结合搜索结果**：用搜索到的竞品和案例激发灵感
4. **允许疯狂想法**：先发散后收敛，不要过早否定

| 时间周期 | 快速验证版 / 长期建设版 |
| 技术路线 | 自研 / 外包 / SaaS / 开源 |
| 商业模式 | 免费 / 付费 / 订阅 / 分成 |
| 目标人群 | 个人 / 小团队 / 企业 / 政府 |
| 地域范围 | 本地 / 全国 / 全球 |
