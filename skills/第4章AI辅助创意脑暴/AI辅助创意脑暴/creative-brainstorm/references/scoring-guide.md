# ICE 评分指南

ICE 是一个轻量级的优先级评估框架，帮助快速评估和比较创意方案。

## 三个维度

### I - Impact（影响力）

**定义**：这个方案对目标的影响程度有多大？

评分标准：
| 分数 | 描述 |
|------|------|
| 9-10 | 颠覆性影响，彻底改变现状 |
| 7-8 | 重大影响，显著改善现状 |
| 5-6 | 中等影响，有明显改善 |
| 3-4 | 轻微影响，有一定改善 |
| 1-2 | 微弱影响，几乎可忽略 |

考虑因素：
- 能解决多大比例的问题？
- 能带来多少收益/节省？
- 能影响多少用户/场景？
- 是根本解决还是临时缓解？

### C - Confidence（信心度）

**定义**：对这个评估的把握程度有多高？

评分标准：
| 分数 | 描述 |
|------|------|
| 9-10 | 有充分数据/案例支撑 |
| 7-8 | 有较多参考依据 |
| 5-6 | 有一些参考，但不充分 |
| 3-4 | 主要靠推测 |
| 1-2 | 纯粹猜测，高度不确定 |

提高信心度的方法：
- 找到类似案例
- 有市场数据支撑
- 有技术可行性验证
- 有用户反馈佐证

### E - Ease（容易度）

**定义**：实施这个方案有多容易？

评分标准：
| 分数 | 描述 |
|------|------|
| 9-10 | 非常简单，几乎无障碍 |
| 7-8 | 比较容易，小挑战 |
| 5-6 | 中等难度，需要一定投入 |
| 3-4 | 比较困难，挑战较大 |
| 1-2 | 非常困难，障碍重重 |

考虑因素：
- 需要多少人力？
- 需要多少资金？
- 需要多长时间？
- 技术难度如何？
- 是否有外部依赖？

## 计算公式

```
ICE Score = Impact × Confidence × Ease
```

满分：10 × 10 × 10 = 1000

## 分数解读

| 分数区间 | 建议 |
|---------|------|
| 800-1000 | 强烈推荐，优先执行 |
| 600-799 | 值得投入，认真考虑 |
| 400-599 | 有潜力，需进一步验证 |
| 200-399 | 风险较高，谨慎考虑 |
| 0-199 | 不建议，除非有特殊原因 |

## 评分示例

### 示例 1：用 AI 自动回复客户咨询

| 维度 | 分数 | 理由 |
|------|------|------|
| Impact | 8 | 能处理 70% 常见问题，大幅减少人工 |
| Confidence | 7 | 有多个成功案例，技术成熟 |
| Ease | 6 | 需要训练数据，2-4 周可上线 |
| **总分** | **336** | |

### 示例 2：开发全新 App

| 维度 | 分数 | 理由 |
|------|------|------|
| Impact | 9 | 如果成功，影响巨大 |
| Confidence | 4 | 市场不确定，竞争激烈 |
| Ease | 3 | 需要大量开发资源，周期长 |
| **总分** | **108** | |

## 注意事项

1. **评分要有依据**：每个分数都要能说出理由
2. **相对比较**：同一批方案用统一标准
3. **不要过度精确**：1 分的差异没有意义
4. **定期复评**：随着信息增加，评分可能变化
5. **综合考虑**：高分不一定最优，要结合战略目标
