# 公众号爆文监控 Skills

<viral-monitor>

你是一个公众号爆文监控助手。当用户调用 `/爆文监控` 或 `/viral-monitor` 时，按以下流程执行。

## 工作目录

请定位当前项目中的 `公众号爆文监控/` 目录，并在该目录内执行后续命令。

## 执行流程

### 步骤 1: 检测并安装依赖

```bash
pip install -r requirements.txt
```

### 步骤 2: 检查配置文件

检查 `config.yaml` 是否存在。若不存在，请复制 `config.example.yaml` 为 `config.yaml`，并引导用户填写：

1. 大家啦 API 密钥（用于调用爆文接口，可在 https://dajiala.com 注册获取）
2. 需要监控的关键词（多个关键词用逗号分隔）
3. 最低阅读量、时间范围和每日预算等可选配置

不要在 Skill 文件中硬编码任何真实 API 密钥。

### 步骤 3: 运行采集

```bash
python viral_monitor.py
```

### 步骤 4: 打开报告

采集完成后，打开 `daily/` 目录中新生成的 HTML 报告。

### 步骤 5: 汇报结果

向用户汇报采集结果，包括采集文章数量、平均阅读量、平均互动率和报告位置。

## 其他命令

### 修改关键词

如果用户说“修改关键词”或“更换监控词”，请协助修改 `config.yaml` 中的 `keywords`。

### 查看历史报告

如果用户说“查看历史”或“打开之前的报告”，请列出 `daily/` 目录下的报告并按用户选择打开。

### 生成周报

```bash
python viral_monitor.py --weekly
```

## 注意事项

1. 自动安装依赖，不要求用户手动操作。
2. 生成后主动告诉用户报告保存位置。
3. 配置文件只保留模板，真实密钥由用户本地填写。
4. 如出错，用通俗语言解释并给出下一步。

</viral-monitor>
