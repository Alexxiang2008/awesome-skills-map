# 【案例】App Store 评价分析 Skills —— 500 条用户评论，1 分钟出报告

- **输入**：App Store 应用 ID 或 URL
- **输出**：情感分析报告 + 改进建议 + 可视化 HTML + CSV 数据

---

## 痛点：产品经理的日常困扰

你有没有这样的经历？

想了解竞品或自家 App 的用户反馈，打开 App Store 一条条翻评论，翻了半小时，笔记本上记了一堆零散的吐槽，最后还是理不出头绪……

传统方式的问题：

| 痛点 | 具体表现 |
|------|----------|
| **效率低** | 手动翻阅评论，500 条要看 2 小时 |
| **难归类** | "闪退"、"崩溃"、"卡死" 其实是同一个问题 |
| **无法量化** | 到底多少人抱怨这个问题？说不清 |
| **缺乏洞察** | 看完一堆差评，不知道该先改哪个 |

---

## 解决方案：让 AI 帮你读评论

这个 Skills 用 Claude 的语义理解能力，自动完成：

```
输入应用 ID → 抓取 500 条评论 → AI 情感分析 → 问题归类 → 生成报告
     ↓              ↓              ↓            ↓           ↓
   1 秒           10 秒          自动         自动      可视化 HTML
```

### 核心能力

1. **自动抓取**：调用 iTunes RSS API，最多获取 500 条最新评论
2. **情感分析**：自动分类正面/负面/中性评价
3. **问题聚类**：AI 识别高频问题，按严重程度排序
4. **改进建议**：基于真实用户反馈，生成 P0/P1/P2 优先级建议
5. **可视化报告**：生成精美的 HTML 报告，支持搜索、过滤、导出

## 技术原理

### 数据来源

Apple 官方 iTunes RSS API（公开接口，无需认证）：

```
https://itunes.apple.com/cn/rss/customerreviews/id=APP_ID/page=1/json
```

### AI 分析

- **情感分类**：基于评分（4-5星正面，1-2星负面，3星中性）
- **问题归类**：Claude 分析负面评论内容，识别高频主题
- **改进建议**：根据问题严重程度和提及次数，生成优先级建议

## 常见问题

### Q: 为什么只能获取 500 条评论？

A: 这是 iTunes RSS API 的限制，最多返回 10 页 × 50 条 = 500 条最新评论。

### Q: 支持 Google Play 吗？

A: 目前只支持 App Store。Google Play 的 API 需要认证，后续可以扩展。

### Q: 报告里的改进建议准确吗？

A: 建议是 AI 基于评论内容动态生成的，准确度取决于评论质量。建议结合实际情况判断。
