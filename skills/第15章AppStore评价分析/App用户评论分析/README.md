# App Store 评价分析器 - 安装说明

## 📦 Skill 简介

自动抓取并分析 App Store 应用的用户评价，发现产品改进机会点。

**核心功能:**
- 📊 自动抓取最多 500 条真实用户评论
- 🎯 智能情感分析（正面/负面/中性）
- 📈 评分分布统计
- 🔍 提取 Top 5 好评和差评
- 💡 生成具体改进建议
- 💾 导出 CSV 格式数据

## 🚀 快速安装

### 方法一：手动安装（推荐）

1. **找到 Claude Code skills 目录**
   ```bash
   # macOS/Linux
   cd ~/.claude/skills

   # Windows
   cd %USERPROFILE%\.claude\skills
   ```

2. **复制 skill 文件**
   ```bash
   # 从当前目录复制
   cp app-review.md ~/.claude/skills/
   ```

3. **验证安装**
   ```bash
   # 在 Claude Code 中运行
   /help
   ```

   应该能看到 `app-review` 或 `评价分析` 出现在 skills 列表中。

### 方法二：直接使用（无需安装）

如果不想安装，可以直接在 Claude Code 中使用：

```
请按照 app-review.md 文件中的指令，帮我分析应用 ID 6448311069 的用户评价
```

## 🔧 技术原理

### API 来源

使用 Apple iTunes RSS API 获取评论数据：

```
https://itunes.apple.com/[region]/rss/customerreviews/id=[APP_ID]/page=[1-10]/json
```

**参数说明:**
- `region`: 地区代码（cn, us, jp 等）
- `APP_ID`: 应用的唯一标识
- `page`: 页码（1-10），每页最多 50 条

### 数据限制

- 最多获取 10 页 × 50 条 = 500 条最新评论
- 只能获取公开的用户评论
- 不同地区的评论需要分别抓取

### 分析方法

1. **情感分类**: 基于评分（4-5星=正面，3星=中性，1-2星=负面）
2. **主题提取**: 使用 AI 分析评论内容，识别高频主题
3. **问题识别**: 从负面评论中提取具体问题和改进建议

## 🎯 最佳实践

### 1. 定期监控
建议每周或每月重新分析，跟踪问题改善情况：

```bash
# 第一次分析
/app-review
> 6448311069

# 一周后再次分析
/app-review
> 6448311069
```

对比两次分析结果，查看：
- 评分是否提升
- 负面评论是否减少
- 新出现的问题

### 2. 竞品对比
同时分析自己的应用和竞品：

```bash
# 分析自己的应用
/app-review
> [你的应用ID]

# 分析竞品
/app-review
> [竞品应用ID]
```

对比分析：
- 用户喜欢竞品的哪些功能
- 竞品存在哪些问题
- 差异化机会在哪里

### 3. 多地区分析
如果应用在多个地区上线，分别分析：

```bash
# 中国区
/app-review
> 6448311069, cn

# 美国区
/app-review
> 6448311069, us
```

不同地区的用户可能有不同的需求和问题。

### 4. 深度跟进
对于高频问题，进行深度分析：

1. 导出 CSV 文件
2. 筛选相关评论
3. 进行用户访谈
4. 制定改进方案

## ⚠️ 注意事项

### 隐私保护
- 用户名会自动脱敏处理（如: j***n）
- 不要在公开场合分享包含用户信息的 CSV 文件

### API 限制
- iTunes RSS API 有访问频率限制
- 建议不要频繁抓取同一应用
- 如遇到限流，请稍后重试

### 数据准确性
- 只能获取最新 500 条评论，不是全部评论
- 评论可能包含多语言，需要人工复核
- AI 分析结果仅供参考，需要结合实际情况判断
