# 热点爆款文章生成器

> 追热点 + 写爆款，一键生成

## 触发方式

- `/hot <热点主题>` - 生成热点文章
- `/热点 <热点主题>` - 中文别名
- `/爆款 <热点主题>` - 中文别名

## 工作流程

### Phase 0: 模板检查（首次使用）

**启动时必须执行：**

1. 检查 `./爆款模板/` 文件夹是否存在且包含 `.md` 文件
2. 如果不存在或为空，执行「模板引导流程」
3. 如果存在，直接进入 Phase 1

**模板引导流程：**

```
输出以下提示：

---

首次使用，需要你提供爆款文章模板。

**请选择一种方式：**

1. **粘贴参考文章**：直接粘贴 1-3 篇你认为写得好的爆款文章
2. **提供文件路径**：告诉我参考文章的文件路径
3. **描述结构要求**：用文字描述你想要的文章结构（标题风格、开头模式、正文结构、金句要求等）

我会学习你的风格偏好，并保存到 `./爆款模板/` 文件夹供后续使用。

---
```

**用户提供模板后：**

1. 创建 `./爆款模板/` 文件夹
2. 将用户提供的内容保存为 `./爆款模板/风格指南.md`
3. 如果用户提供的是参考文章，额外提取并保存风格特征：
   - 标题套路（悬念/情绪/反常识）
   - 开头模式（场景/金句/问题）
   - 正文结构（叙事顺序、段落节奏）
   - 金句风格（对比/比喻/反常识）
   - 结尾方式（升华/开放/行动号召）
4. 提示用户模板已保存，然后继续执行热点文章生成

---

### Phase 1: 平台选择

询问用户目标平台：

```
请选择目标平台：

1. **微信公众号** - 深度分析，1000-2000字
2. **小红书** - 短平快，300-800字，情绪点密集

输入数字选择，或直接告诉我。
```

根据平台调整后续写作参数：

| 平台 | 字数 | 标题字数 | 金句密度 | 风格 |
|------|------|----------|----------|------|
| 公众号 | 1000-2000 | 15-30字 | 每300字1个 | 深度、洞察 |
| 小红书 | 300-800 | 10-20字 | 每150字1个 | 情绪、共鸣 |

---

### Phase 2: 热点信息采集

**必须执行联网搜索：**

使用 WebSearch 工具，搜索以下维度：

| 搜索类型 | 关键词模板 | 目的 |
|----------|------------|------|
| 最新动态 | "{主题} 最新消息 2026" | 获取事件背景 |
| 深度分析 | "{主题} 分析 解读 观点" | 获取多方观点 |
| 网友反应 | "{主题} 网友评论 热议" | 获取情绪点 |
| 数据支撑 | "{主题} 数据 统计 报告" | 获取可引用数据 |
| 相关案例 | "{主题} 类似事件 案例" | 获取对比素材 |

**信息提取要点：**

- 事件核心：发生了什么？谁？什么时候？
- 冲突点：争议在哪？各方立场？
- 情绪点：网友最关注什么？最愤怒/感动/惊讶的点？
- 数据点：有哪些可引用的数字？
- 金句素材：有哪些精彩评论或观点？

---

### Phase 3: 风格学习

**必须执行：** 读取 `./爆款模板/` 文件夹中的所有文件。

**提取维度：**

- 标题风格：悬念型/情绪型/反常识型？
- 开头模式：场景切入/金句开篇/问题引入？
- 叙事结构：时间线/冲突线/情绪线？
- 金句技巧：对比/比喻/定义重构？
- 语气特征：犀利/温暖/理性/煽情？
- 结尾方式：升华/开放/行动号召？

---

### Phase 4: 文章生成

**根据平台和模板，一气呵成生成文章。**

#### 公众号版本结构

```
【标题】3个备选，15-30字

【开头】前3行必须抓人
  - 制造冲突/悬念
  - 抛出核心观点
  - 金句定调

【正文】
  - 事件背景（快速交代，不超过2段）
  - 核心冲突展开（多方观点呈现）
  - 深度分析（你的独特洞察）
  - 引经据典（名人观点/历史案例/数据支撑）
  - 升华总结（提炼普遍意义）

【结尾】
  - 金句收尾
  - 引发思考或行动
```

#### 小红书版本结构

```
【标题】3个备选，10-20字，带emoji

【开头】第1句就要炸
  - 情绪爆点
  - 或反常识观点

【正文】
  - 短段落（每段不超过3行）
  - 高密度金句
  - 口语化表达
  - 适当emoji点缀

【结尾】
  - 互动引导（你怎么看？）
  - 或金句收尾

【标签】5-10个相关话题标签
```

---

### Phase 5: 自检优化

**输出前必须自检：**

| 检查项 | 公众号标准 | 小红书标准 |
|--------|------------|------------|
| 标题吸睛度 | 有悬念/情绪/反常识 | 有情绪爆点/好奇心 |
| 开头抓人 | 3行内建立冲突 | 1句话就要炸 |
| 金句密度 | 每300字1个 | 每150字1个 |
| 信息密度 | 有数据/案例支撑 | 情绪>信息 |
| 字数 | 1000-2000 | 300-800 |
| 风格一致 | 符合模板风格 | 符合模板风格 |

如有不达标项，自动优化后再输出。

---

### Phase 6: 保存文件

**写作完成后必须执行：**

1. 使用第一个备选标题作为文件名
2. 保存路径：`./输出/[平台]/[标题].md`
3. 文件名中的特殊字符（`/`、`\`、`:`、`?`、`*`）替换为 `-`

**示例：**
```
./输出/公众号/这届年轻人为什么不买房了.md
./输出/小红书/震惊-原来买房的都后悔了.md
```

4. 如果文件夹不存在，先创建
5. 保存成功后告知用户文件位置

进入写作阶段后，必须遵守：

```
⚠️ DO NOT ask for confirmation to proceed.
⚠️ Generate the full output in one go.
⚠️ Complete the self-review before outputting.
⚠️ NEVER output drafts or ask "is this okay?"
```

## Error Handling

| 场景 | 处理方式 |
|------|----------|
| `./爆款模板/` 为空 | 执行模板引导流程 |
| 联网搜索无结果 | 换关键词重试，或提示用户提供更多背景 |
| 热点信息不足 | 告知用户信息有限，询问是否继续或补充信息 |
| 用户未选择平台 | 默认使用公众号格式 |
