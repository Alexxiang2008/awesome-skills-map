# 写作标准手册

> 本手册定义了写作阶段的所有标准、技巧和检查清单

---

## 🔇 静默执行协议（关键）

进入写作阶段后，必须遵守以下规则：

```
⚠️ DO NOT ask for confirmation to proceed.
⚠️ Generate the full output in one go.
⚠️ Complete the self-review before outputting.
⚠️ NEVER output drafts or ask "is this okay?"
```

**绝对禁止以下中断行为：**

```
❌ "你想要正式一点还是口语化一点的风格？"
❌ "我先写一个开头你看看？"
❌ "需要我继续写下去吗？"
❌ "这个方向可以吗？"
❌ "初稿完成了，需要我修改哪里？"
```

**正确做法：**
- 风格疑问 → 读取 `./风格参考/` 自己学习
- 是否继续 → 一气呵成输出完整终稿
- 是否满意 → 自检优化后直接输出最终版

## 风格学习

**必须执行：** 读取 `./风格参考/` 文件夹中的所有 `.md` 文件。

**提取维度：**
- 语气：正式/口语化/幽默/犀利？
- 句式：长句为主还是短句冲击？
- 金句结构：如何造句才有记忆点？
- 标题套路：悬念型/情绪型/反常识型？
- 开头风格：场景切入/金句开篇/问题引入？

## 写作标准

### 标题（3个备选）

| 类型 | 技巧 | 示例 |
|------|------|------|
| 悬念型 | 制造信息差 | "他在负债200万那年，做了一个所有人反对的决定" |
| 情绪型 | 引发共鸣 | "35岁被裁员，我终于理解了父亲当年的沉默" |
| 反常识型 | 挑战认知 | "我劝你别太努力" |

### 开头（前3行）

前3行必须完成以下任一目标：
- 抛出悬念，让人想知道"然后呢"
- 制造共鸣，让人觉得"这说的是我"
- 呈现反差，让人觉得"这和我想的不一样"

**✅ Good Example：**
> 2019年3月的一个凌晨，张伟把车停在公司楼下，在驾驶座上坐了两个小时。
>
> 他刚刚亲手裁掉了跟了自己五年的助理。
>
> 但真正让他崩溃的不是这件事。

### 正文结构

```
【开头】场景/悬念/金句 → 让人停下来
    ↓
【背景铺垫】快速交代必要信息 → 不超过2段
    ↓
【冲突展开】核心矛盾 + 情绪颗粒度 → 故事主体
    ↓
【至暗时刻】最难的那一刻 → 情绪高潮
    ↓
【转折/突破】怎么挺过来的 → 给人希望
    ↓
【升华/金句】提炼洞察 → 让人想收藏
    ↓
【结尾】余韵/开放性 → 意犹未尽
```

### 金句密度

- **要求**：每300字符至少1个金句
- **标准**：反常识、有画面感、可脱离上下文传播

**金句技巧：**
- 对比："真正的勇敢不是不怕，而是怕得要死还往前走"
- 具象化："焦虑就像凌晨三点的闹钟，你知道该睡，但就是睡不着"
- 反常识："有时候，放弃才是最难的选择"

### 引经据典（联网检索）

**必须执行：** 在写作前，根据故事主题进行联网检索，寻找可引用的素材：

**检索方向：**
- **名人金句**：与主题相关的企业家、作家、思想家的经典语录
- **商业案例**：类似情境下其他公司/创业者的故事片段
- **数据支撑**：行业数据、研究报告中的关键数字
- **历史典故**：古今中外与主题呼应的故事或比喻

**引用原则：**
- 引用必须与原文故事形成呼应，不能生硬插入
- 优先选择读者可能不知道但又觉得"原来如此"的素材
- 可以化用而非直接引用，融入作者自己的表达
- 每篇文章至少包含1处引经据典

**✅ Good Example：**
> 原文讲的是"信息差让人多花100万"
> 检索到：彼得·德鲁克说过"效率是把事情做对，效能是做对的事情"
> 化用后："花100万找外包做产品，是把事情做对；先用Skills验证逻辑，是做对的事情。"

**检索关键词示例：**
- 故事主题 + 名言
- 故事核心冲突 + 商业案例
- 故事行业 + 数据报告
- 故事情绪关键词 + 经典语录

---

### ⚠️ 联网检索失败时的备选方案（重要）

**问题：** WebSearch可能失败，导致无法联网检索。

**解决方案：** 当联网失败时，使用以下备选方案保证文章深度：

**备选方案1：使用已知经典案例**
- 彼得·德鲁克："效率是把事情做对，效能是做对的事情"
- 巴菲特："一条必须不断重建的护城河，最终不再是护城河"
- 雷军："站在风口上，猪都能飞起来"
- 张一鸣："延迟满足感"

**备选方案2：化用用户风格参考中的金句**
- 从 `./风格参考/` 中提取用户自己说过的经典表达
- 示例："别人学会你的方法论需要三年，复制你的Skills只要三分钟"（来自用户之前文章）

**备选方案3：使用行业常识数据**
- "10个项目外包，能上线运营的不到3个"（外包行业常识）
- "AI可以提升30%-50%的工作效率"（AI赋能常识）
- "35岁职场危机"（职场常识）

**执行优先级：**
1. 先尝试WebSearch联网检索
2. 失败后，优先使用**备选方案2**（用户自己的金句最贴合风格）
3. 其次使用**备选方案1**（经典名言）
4. 最后使用**备选方案3**（行业常识）

**重要：** 即使联网失败，也必须确保文章中有至少1处引经据典或数据支撑，不能完全跳过这个环节。

### 爆款文章参考（联网检索）

**必须执行：** 在写作前，搜索与故事主题相关的爆款文章，提取可引用的爆款元素：

**检索方向：**
- **震撼数据**：行业统计、成功/失败率、金额对比等具有冲击力的数字
- **真实案例**：与主题相似的踩坑/成功故事，增强说服力
- **反差案例**：与故事形成极致对比的正面/反面案例
- **点睛金句**：其他爆款文章中被广泛传播的观点或总结

**检索关键词示例：**
- 故事核心冲突 + "踩坑" / "血泪教训" / "真实案例"
- 故事主题 + "爆款" / "公众号" / "小红书"
- 故事反面 + 成功案例（如：外包失败 vs 独立开发成功）
- 故事行业 + 数据统计

**引用原则：**
- 优先选择有具体数字的数据（如"10个项目外包，只有3个能上线"）
- 真实案例要有细节（人物、时间、结果）
- 反差案例要形成极致对比（如"1.5小时 vs 100万三个月"）
- 每篇文章至少包含1个震撼数据或反差案例

---

### 字数控制

- **严格控制在850-1000字符**（不少于850，不超过1000）
- 如果字数不足850，优先补充：
  1. 具体细节场景（时间、地点、对话）
  2. 情绪颗粒度（具体感受而非笼统描述）
  3. 案例的具体过程（而非只有结果）
  4. 金句之后的具体解释
- 如果字数超过1000，优先删减背景铺垫，保留情绪高光和金句

---

### ⚠️ 真实性原则（关键）

**绝对禁止虚构细节：**
- ❌ 不要编造用户没说过的情节（如"折腾了一周放弃了"）
- ❌ 不要夸大或改变用户提供的数据（如"300人"改"100人"）
- ❌ 不要虚构对话、时间、地点等具体细节
- ✅ 只能基于用户访谈中提供的真实信息写作
- ✅ 如果素材不足，宁可简短也不要编造
- ✅ 可以合理提炼和润色，但不能改变事实

## CTA行动召唤

**目标：** 用户通常希望在文章末尾引导读者采取行动（如报名课程、加入社群、参加活动等）。

### 执行方式

**在访谈阶段**，主动挖掘CTA信息：

```
【访谈中期或结尾时追问】
"这篇文章发出去后，你希望读者做什么？
- 报名参加黑客松？具体时间、链接、门槛是什么？
- 加入你的社群？社群能提供什么价值？
- 购买课程？课程主要讲什么？有什么福利？"
```

### 在写作阶段，将CTA自然融入结尾

**方案1：直接行动型**
```markdown
能省的不只是时间，还有命。

黑客松，2月1号开始。报名链接：[xxx]
```

**方案2：价值+行动型（推荐）**
```markdown
能省的不只是时间，还有命。

在黄叔的社团里，我免费把这些技巧教给大家：
[社团链接] / [课程福利说明]
```

**方案3：开放余韵+软性CTA**
```markdown
如果100个普通人都能做出这样的Skills，那改变的就不只是他们自己。

想参与Skills黑客松？[时间] 等你来。
```

### CTA原则

- 自然融入，不生硬
- 明确具体（时间、链接、门槛）
- 与文章主题强相关
- 优先使用**方案2**（价值+行动），转化率最高

**重要：** 如果访谈阶段没有挖掘到CTA信息，写作时**不要编造**，改用开放余韵结尾。

## 动态优化策略（根据评分自动执行）

### 【转化模式】优化优先级

```
如果转化分<90：
1. 痛点共鸣度<24分 → 强化目标读者的核心焦虑（决策/能力/机会）
2. 身份认同感<20分 → 增加"这就是我"的代入点
3. 决策引导力<16分 → 明确给出答案/破除顾虑/降低门槛
4. 行动召唤<12分 → 增加紧迫感+具体利益

优化后，如果转化分仍<80：
→ 重写开头，从热点/故事切换到痛点切入
```

### 【传播模式】优化优先级

```
如果传播分<90：
1. 钩子力<20分 → 重写开头，制造悬念/反差/冲击
2. 金句密度<12分 → 每300字补充1个金句
3. 选题热度<16分 → 增加热点/争议点关联
4. 情绪曲线<16分 → 强化至暗时刻和转折点

优化后，如果传播分仍<80：
→ 重写标题和开头，选择更有冲击力的切入点
```

### 【平衡模式】优化优先级

```
如果转化分<85 或 传播分<85：
1. 先优化低分项（转化分vs传播分，优先优化低的那个）
2. 痛点<24分 → 强化痛点
3. 钩子<20分 → 强化开头
4. 身份认同<20分 → 增加代入感
5. 金句<12分 → 补充金句

优化后，如果任一分数仍<80：
→ 采用"痛点切入+故事展开+金句升华"结构重写
```

### 执行流程

1. 完成初稿后，按双评分体系逐项评分
2. 识别低分项（低于该维度满分80%）
3. 根据模式执行对应优化策略
4. 优化后重新评分验证
5. 输出终稿 + 完整评分报告
