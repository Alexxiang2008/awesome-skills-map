## ✅ Good Cases

### 案例1：一步到位的访谈与写作

**用户输入：** `/interview 我的第一次创业失败`

**AI（开场）：**
> 创业失败这件事，你是在哪个瞬间意识到"完了"的？

**用户：** 合伙人在融资前一周突然说不干了。

**AI（追问）：**
> 他说这话的时候，你们是在哪里？你当时第一反应是什么——愤怒、慌张、还是一种奇怪的平静？

*[继续5-8轮深度访谈...]*

**AI（判断素材充足，进入写作）：**
> 我觉得素材已经足够丰富了。现在我来帮你写成文章——请稍等，我先学习一下你的风格参考，然后一气呵成输出终稿。

*[读取风格参考 → 生成初稿 → 自检优化 → 爆款观察员审核 → 输出终稿]*

**输出结果：**

```markdown
## 备选标题

1. 融资前7天，合伙人说他不干了
2. 我花了3年才明白：散伙比失败更痛
3. 那天我在会议室坐到凌晨，等一个不会来的人

---

## 正文

2021年4月的一个周五下午，李明在公司会议室里等了三个小时。

他在等合伙人张涛。融资的事下周就要签字，所有材料都准备好了。但张涛的微信，已经12小时没有回复。

[...正文继续，包含情绪颗粒度、细节场景、金句升华...]

后来有人问我，创业最难的是什么。我说，不是没钱，不是没客户，是你以为和你一起扛的那个人，突然不见了。

那种感觉，比失败更让人窒息。
```

### 案例2：优秀开头示例

**示例1：场景切入 + 悬念**
> 数十亿美金。100倍。
>
> 凌晨两点半，我盯着这条新闻，翻来覆去睡不着。
>
> 不是因为兴奋，是因为恐惧。

**示例2：反差切入**
> 我亲手把自己的饭碗，复制了几百份发出去。
>
> 朋友说我疯了。
>
> 但我知道，这是唯一的活路。

**示例3：痛点切入**
> 做了一年Skills,花了至少200小时。
> 真正每天用的,只有2个。
> 其他125个?全在吃灰。

**对比型：**

## ❌ Anti-Patterns

### 反面案例1：禁止的中断行为

**❌ 错误做法：**

```
AI: "我先写一个开头你看看？"
AI: "你想要正式一点还是口语化一点的风格？"
AI: "需要我继续写下去吗？"
AI: "这个方向可以吗？"
AI: "初稿完成了，需要我修改哪里？"
```

**✅ 正确做法：**

```
AI: "好的，主线确定了。我觉得素材已经足够丰富了。
     现在我来帮你写成文章——请稍等，我先学习一下你的风格参考，然后一气呵成输出终稿。"

[直接输出完整终稿，包括：备选标题 + 正文 + 爆款评分报告]
```

---

### 反面案例2：虚构细节

**❌ 错误做法：**

```
访谈中用户说："我当时很难过"

AI写作时编造：
"凌晨3点，他坐在办公室里，看着窗外的霓虹灯，一根接一根地抽烟。
桌上的咖啡已经凉了，电脑屏幕上还停留在那封辞职信。"

→ 这些细节用户都没说过，属于虚构！
```

**✅ 正确做法：**

```
访谈中用户说："我当时很难过"

AI追问："能描述一下当时的场景吗？你在哪里？在做什么？"

用户补充："在办公室，一直坐到很晚"

AI写作时基于真实信息：
"那天晚上，他一直坐在办公室里，直到大楼的灯都暗了。"

→ 基于用户提供的真实信息，适度润色
```

### 反面案例3：开头平淡

**❌ 错误做法：**

```
创业是一件很不容易的事情。
很多人都在创业的路上遇到过困难。
今天我想分享一下我的创业经历。
```

→ 问题：陈词滥调，缺少悬念，读者会直接划走

**✅ 正确做法：**

```
融资前7天，合伙人发来一条微信："我不干了。"

后面是句号，不是感叹号。

这是我创业三年来，看到最平静、也最残酷的六个字。
```

→ 直接切入冲突，制造悬念和情绪

### 反面案例4：金句堆砌

**❌ 错误做法：**

```
成功不是终点，失败不是末日，勇气才是永恒。
真正的强者，不是没有眼泪，而是含着眼泪依然奔跑。
人生就像一场马拉松，重要的不是起点，而是终点。
```

→ 问题：金句太密集，且都是陈词滥调，没有个人特色

**✅ 正确做法：**

```
后来有人问我，创业最难的是什么。

我说，不是没钱，不是没客户，是你以为和你一起扛的那个人，突然不见了。

那种感觉，比失败更让人窒息。
```

→ 金句来自真实感受，有具体场景支撑，可独立传播

## 常见问题 FAQ

### Q1: 什么时候结束访谈进入写作？

**A:** 满足以下条件之一即可进入写作：
1. 达到对应模式的结束条件（见《访谈追问手册》）
2. 用户主动表示"可以开始写了"
3. 访谈超过10轮但素材已充足

### Q2: 如果联网检索失败怎么办？

**A:** 使用备选方案（见《写作标准手册》→ 引经据典章节）：
1. 优先使用用户风格参考中的金句
2. 其次使用经典名言（德鲁克、巴菲特等）
3. 最后使用行业常识数据

**关键：** 即使失败也不能跳过，必须至少包含1处引经据典。

### Q3: 字数不够850怎么办？

**A:** 优先补充以下内容：
1. 具体细节场景（时间、地点、对话）
2. 情绪颗粒度（具体感受而非笼统描述）
3. 案例的具体过程（而非只有结果）
4. 金句之后的具体解释

**禁止：** 不要虚构用户没说过的细节来凑字数

### Q4: 如何判断是否需要智能暂停？

**A:** 满足任一条件就暂停：
1. 已获得基本框架但方向不明确
2. 连续2轮追问同一个细节
3. 用户回答开始重复/模糊
4. 已有3-4轮对话但还没问过文章目标

**暂停后：** 提供三种主线方案，让用户选择方向

### Q5: 评分报告一定要输出吗？

**A:** 是的，必须输出完整的评分报告，这是强制要求。

评分报告包括：
- 转化分和传播分详情
- RPM预测和ROI预测
- 优化建议
- 发布策略建议

**不允许省略任何部分。**
