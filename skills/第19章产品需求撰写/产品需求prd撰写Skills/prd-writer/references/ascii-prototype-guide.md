# ASCII 原型设计指南

## 设计原则

1. **只展示 MVP 功能**：不要把 V2+ 的功能画进原型
2. **清晰表达布局**：用户能理解各区域的功能
3. **标注交互元素**：按钮、输入框、链接等要明确标识
4. **保持简洁**：ASCII 图的目的是传达概念，不是精确设计

## 三种设计理念模板

### 方案 A：极简主义

**特点**：功能最小化，界面最简洁，一屏只做一件事

```
┌─────────────────────────────────┐
│           [产品名称]            │
├─────────────────────────────────┤
│                                 │
│                                 │
│     ┌───────────────────────┐   │
│     │                       │   │
│     │    核心操作区域       │   │
│     │                       │   │
│     └───────────────────────┘   │
│                                 │
│         [[ 主要操作 ]]          │
│                                 │
│                                 │
└─────────────────────────────────┘
```

### 方案 B：功能导向

**特点**：突出核心功能，操作便捷，功能分区明确

```
┌─────────────────────────────────┐
│ ☰  [产品名称]           👤     │
├─────────────────────────────────┤
│ [快捷操作1] [快捷操作2] [更多]  │
├─────────────────────────────────┤
│                                 │
│  ┌─────────┐  ┌─────────┐      │
│  │ 功能卡1 │  │ 功能卡2 │      │
│  └─────────┘  └─────────┘      │
│                                 │
│  ┌─────────────────────────┐   │
│  │     内容展示区域         │   │
│  └─────────────────────────┘   │
│                                 │
├─────────────────────────────────┤
│  🏠    📊    ➕    💬    👤   │
└─────────────────────────────────┘
```

### 方案 C：信息密集

**特点**：一屏展示更多信息，适合专业用户

```
┌─────────────────────────────────────────────────┐
│ [产品名称]  [搜索...]  [筛选▼]  [导出]  👤     │
├──────────────┬──────────────────────────────────┤
│              │                                  │
│  侧边导航    │  ┌─────────────────────────────┐│
│              │  │ 数据表格/列表               ││
│  > 分类1     │  │ ─────────────────────────── ││
│    分类2     │  │ 行1  |  数据  |  数据  |操作││
│    分类3     │  │ 行2  |  数据  |  数据  |操作││
│              │  │ 行3  |  数据  |  数据  |操作││
│  ─────────   │  └─────────────────────────────┘│
│              │                                  │
│  快捷入口    │  ┌──────────┐ ┌──────────┐      │
│  [+新建]     │  │ 统计卡1  │ │ 统计卡2  │      │
│              │  └──────────┘ └──────────┘      │
│              │                                  │
└──────────────┴──────────────────────────────────┘
```

## 交互说明模板

每个原型需要附带交互说明：

```markdown
**交互说明**：

1. **[元素名称]**：[交互描述]
   - 点击后：[结果]
   - 状态变化：[描述]

2. **[元素名称]**：[交互描述]
   - 输入时：[实时反馈]
   - 提交后：[结果]

3. **页面跳转**：
   - [操作] → [目标页面]
```

## 注意事项

1. **保持一致性**：同一套原型使用相同的组件风格
2. **标注优先级**：核心功能要在视觉上突出
3. **考虑响应式**：如果是移动端，用更窄的宽度
4. **预留扩展位**：为 V2 功能预留位置（但不画出来）
